如何愉快使用网易云音乐，解锁变灰歌曲

发布 : 2020-05-03 分类 : 日记 浏览 :

目前听音乐主要使用网易云音乐和虾米音乐，由于好多音乐涉及版权问题无法收听，多个软件使用又比较麻烦，今天无意中发现了这个文章，亲测有效

之前每次遇到这样的歌曲，我都是下载下来收听的

一、网易云音乐下载方法（不适用需要VIP下载的歌曲）

用浏览器随意找一首歌曲，查看地址链接获取到当前歌曲的ID

将歌曲的ID替换下面地址的ID，再用浏览器打开这个地址

然后复制当前浏览器地址，使用下载软件下载

二、在线收听

提供可以测试的服务器地址 端口都是8000

项目地址

1、功能特点

解锁网易云音乐客户端变灰歌曲！
使用网易云旧链 / QQ / 虾米 / 百度 / 酷狗 / 酷我 / 咕咪 / JOOX 音源替换变灰歌曲链接 (默认仅启用前四)
为请求增加 X-Real-IP 参数解锁海外限制，支持指定网易云服务器 IP，支持设置上游 HTTP / HTTPS 代理
完整的流量代理功能 (HTTP / HTTPS)，可直接作为系统代理 (同时支持 PAC)

2、安装部署

服务器部署可以多个终端一起使用，Windows部署只可以单机使用，或者同局域网下通过IP访问，二者选一即可

3、服务器部署（二选一）

https://nodejs.org/dist/v12.16.3 下载对应的安装包
先安装node

下载项目之后 启动即可

4、Windows部署（二选一）

下载安装nodejs （官方地址：https://nodejs.org/en/）
安装完成后检查是否成功
开始-运行-输入cmd-回车
输入node -v 看到输出版本号，说明安装成功
下载项目压缩包 https://github.com/nondanee/UnblockNeteaseMusic/archive/master.zip
解压后进入目录
打开一个终端窗口 npm install

5、启动服务

服务器端和Windows端的命令窗口不能关闭，关闭之后服务就停止了
node app.js -p 8000:8043

6、可以利用一些服务还让他后台运行

pm2（目前还有问题）
forever（待测试）
Screen-linux 后台

三、软件设置

下载安装网易云音乐客户端
点击右上角的【设置】-【工具】-【Http代理】-【自定义代理】-下拉框选择【HTTP代理】
服务器
本机输入 127.0.0.1
局域网可以选择本机的IP地址
端口就是上面的8000 可以自行更改
最后重启客户端，即可收听灰色的歌曲

四、mac客户端使用（亲测有效）

Proxifier 是 mac 下的辅助代理软件，可以指定程序指定域名端口走指定代理协议，比如让系统自带的邮件同步 gmail 邮箱，比如屏蔽某个 app 的联网权限

1、 需要先下载Proxifier mac版本的软件，并激活使用

2、 添加一个目标代理（proxy）
address 127.0.0.1
protocol选择 https
authentication不需要填写

3、 代理规则（rule）
name NeteaseMusic
Applications NeteaseMusic;com.apple.WebKit.Networking

4、 Action 选择上一步添加的 HTTPS 代理

5、 保存

自签证书
新版通过自签证书直接代理 https 流量能够解决此问题 (同时支持 hosts 模式和代理模式)
1、 生成 CA 私钥
openssl genrsa -out ca.key 2048

2、 生成 CA 证书 (“YOURNAME” 处填上你自己的名字)
openssl req -x509 -new -nodes -key ca.key -sha256 -days 1825 -out ca.crt -subj "/C=CN/CN=UnblockNeteaseMusic Root CA/O=YOURNAME"

3、 生成服务器私钥
openssl genrsa -out server.key 2048

4、 生成证书签发请求
openssl req -new -sha256 -key server.key -out server.csr -subj "/C=CN/L=Hangzhou/O=NetEase (Hangzhou) Network Co., Ltd/OU=IT Dept./CN=*.music.163.com"

5、 使用 CA 签发服务器证书
openssl x509 -req -extfile <(printf "extendedKeyUsage=serverAuth\nsubjectAltName=DNS:music.163.com,DNS:*.music.163.com") -sha256 -days 365 -in server.csr -CA ca.crt -CAkey ca.key -CAcreateserial -out server.crt

6、 将服务器私钥 (server.key) 和服务器证书 (server.crt) 拷贝到仓库中覆盖原有文件

7、 最后双击安装 CA 证书，需要手动将证书配置为始终信任

五、 IOS手机端配置（待测试）

此时你的电脑或者 VPS 作为服务端，你的 iOS 设备是客户端

1、 安装服务端目录里 ca.crt 证书，需要在 iOS 设置-通用-描述文件与设备管理，安装 UnblockNeteaseMusic Root CA 证书。然后回到 通用-关于本机-证书信任设置，勾选启用刚刚添加的证书

2、 添加节点，类型选择 HTTP（注意不是HTTPS）服务器填写服务端的 IP（你作为服务器的电脑的局域网地址或者你的 VPS 的 IP/域名），端口填写你启动服务的端口号，就是 port1 ，填写一个备注名比如 UnblockNetease

3、 编辑配置，点击本地文件 -> default.conf（你当前使用的配置文件）-> 编辑配置 -> 添加规则，类型 USER-AGENT 选项 UNBLOCKNETEASE 用户代理 NeteaseMusic*

添加下面三条规则，选项都选择刚添加的 UnblockNetease 节点

4、 保存并启动
