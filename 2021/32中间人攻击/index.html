32中间人攻击

发布 : 2021-09-11 分类 : 网络安全 浏览 :

中间人攻击

中间人攻击原理

中间人攻击（Man-in-the-MiddleAttack，简称“MITM 攻击”）是一种”间接”的入侵攻击，这种攻击模式是通过各种技术手段 将受入侵者控制的一台计算机虚拟放置在网络连接中的两台通信计算机之间，这台计算机就称为”中间人”

ARP解析过程 Address Resolution Protocol

地址解析协议，是根据IP地址获取物理地址的一个TCP/IP协议，主机发送信息时 将包含目标IP地址的ARP请求广播到网络上的所有主机，并接收返回消息，以此确定目标的物理地址；收到返回消息后将该IP地址和物理地址存入本机ARP缓存中并保留一定时间，下次请求时直接查询ARP 缓存以节约资源

缺点：
ARP是基于传闻的协议工作，不仅会接收到本机发请求出去的返回响应包，还会接收本机没有请求的响应包，接收后并会更新本地ARP缓存，正是因为ARP有了这样的缺点，所以很容易被人利用ARP欺骗

使用arpspoof实施中间人攻击

Arpspoof是一个非常好的ARP欺骗的源代码程序。它的运行不会影响整个网络的通信，该工具通过替换传输中的数据从而达到对目标的欺骗

流程
让目标认为kali主机就是网关，但是为了能够实现目标机器被欺骗的过程中，还能正常上网，需要做双向欺骗，所以在kali主机上开启转发功能，把流量都引进kali主机，然后再kali主机上对引入的流量进行嗅探

临时开启路由转发
echo 1 > /proc/sys/net/ipv4/ip_forward

对客户机实施欺骗
arpspoof -t 192.168.1.10 -r 192.168.1.1
-t 指定欺骗的主机IP地址
-r 指定路由的IP地址

最终实现了对客户端欺骗为网关，对网关欺骗为客户端，从而实现中间人欺骗，在命令启动后，会不停的自动发数据欺骗数据包
由于在kali主机上开启了路由转发功能，客户端访问请求会发到kali主机上，然后由kali转发到真实服务器上，然后服务器会把响应发回给kali主机，最后由kali主机转发给客户端，这样以来，所有的流量都
会经过kali主机，那么后续的操作我们只需要要在kali主机上进行嗅探即可

在客户端上查看ARP信息，发现网关的MAC和kali的MAC一样的

利用中间人攻击还原目标机访问的图片

sudo driftnet -i eth0
mkdir images
driftnet -i eth0 -a -d images -s

此时执行命令就不会出现预览窗口，而是嗅探到的图片直接保存到 images 目录下

利用中间人攻击抓取目标机提交的密码

Dsniff是一个著名的网络嗅探工具包、高级口令嗅探工具、综合性的网络嗅探工具包

dsniff -i eth0 -m
-i 指定嗅探网口
-m 表示自动启用自动协议检测

dsniff支持的协议
cat /usr/lib/x86_64-linux-gnu/dsniff.services

使用kali Ettercap工具实现中间人攻击

应用程序 => 嗅探/欺骗 => Ettercap graphical
点击选择栏中的”Sniff”而后点击”Unified sniffing”
选择嗅探的网卡接口
开始扫描主机：点击选项栏中的”Hosts”而后点击”scan for hosts”
扫描完成后会自动生成主机列表
查看主机列表，点击”Hosts”而后点击”Hosts list”
查看被扫描的主机
