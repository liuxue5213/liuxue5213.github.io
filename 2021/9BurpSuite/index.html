9BurpSuite

发布 : 2021-05-01 分类 : Linux 浏览 :

一、 mac 安装破解版

破解包 https://github.com/TrojanAZhen/BurpSuitePro-2.1

1. 先安装jdk

查看是否安装jdk10
ls /Library/Java/JavaVirtualMachines/

如果有10版本的JDK需要先删了，删除命令
sudo rm -rf /Library/Java/JavaVirtualMachines/jdk-10.0.2.jdk

在Finder中删除JavaAppletPlugin.plugin和JavaControlPanel.prefPane文件
/Library/Internet\ Plug-Ins/JavaAppletPlugin.plugin
/Library/PreferencesPanes/JavaControlPanel.prefpane

2. 下载、安装BurpSuite

可以从官网下载-推荐参考github文章的下载链接

3. 破解

在【访达->应用程序】中找到 burp suite，右键显示包内容，进入/Contents/java/app，然后把下载好包Burp_Suite_Pro_v1.7.37_Loader_Keygen_macbook.zip里的两个工具拉进来；
然后返回，再在/Contents/vmoptions.txt中添加

启动bp软件

启动注册机

复制License 内容，另存为一个txt文件 license.txt（名称可自定义）

从bp点击Select license key file，选择刚刚保存的license.txt

然后下一步，点击Manual activation，复制选项2里面的请求码到注册机，run生成激活码

复制激活码到bp，即可进入，最后可以从help => license 查看激活日期

启动bp，需要从命令行执行，或者进入复制破解包的目录，点击burpsuite_pro.jar启动

4. 参考页面

https://www.sqlsec.com/2019/11/macbp.html#toc-heading-4
https://blog.csdn.net/whowhowhoisimportant/article/details/107541349

BP基础介绍

Proxy 模块

Intercept：截断；用于显示和修改Http请求和相应，通过你的浏览器和web服务器之间拦截流量实现功能，在Proxy的选项中，可以配置拦截规则来确定是什么请求和相应被拦截（例如，范围内的项目，特定文件扩展名、项目要求与参数）
forword：将请求包发出
drop：丢掉请求包
intercept is on/off：截断的开关
Action：对请求包的一些操作，我们对包体的任何修改都可以在这个窗口内进行直接修改，然后发送
Raw：常规数据包模式
header：请求头
Hex：十六进制数据

HTTP history:凡是访问过的网站，都会被记录下来；
optons：设置，监听 8080 端口

浏览器，设置8080端口代理；

手机使用

optons => Proxy Listeners => Add specific address 选择本机ip => ok

手机安装证书

import / export CA certificate => export => certificate in DER format => 选择保存地址 => ok

可以勾选 “intercept requests based on the following rules” 基于以下规则拦截请求，进行过滤，不抓取图片之类的信息

Target 模块

site map / scope 记录目标，方便后期扫描使用

自动：爬网+扫描 （在工具中指定一个 URL）
代理：代理+扫描（浏览器去访问页面，并把访问的请求交给代理）

scope（范围）：深黑色显示抓包的站点，灰色的是蜘蛛爬出来的，没有抓包，设置 scope 可以过滤灰色的信息；

show only in-scope items:只显示在 scope 里面的；
show only requested items:真正发生过请求的，（黑色的）；
show only parameterized requests:只显示有参数的页面，想一下要是不提交数据，基本上是不会有安全隐患的；

spider 模块

scanner 模块，只有高级版有扫描功能

intruder 模块，攻击模块

Burp Intruder是一个强大的工具，用于自动对Web应用程序自定义的攻击，Burp Intruder 是高度可配置的，并被用来在广范围内进行自动化攻击。你可以使用 Burp Intruder 方便地执行许多任务，包括枚举标识符，获取有用数据，漏洞模糊测试。合适的攻击类型取决于应用程序的情况，可能包括：缺陷测试：SQL 注入，跨站点脚本，缓冲区溢出，路径遍历；暴力攻击认证系统；枚举；操纵参数；拖出隐藏的内容和功能；会话令牌测序和会话劫持；数据挖掘；并发攻击；应用层的拒绝服务式攻击

1：Target 用于配置目标服务器进行攻击的详细信息。
2：Positions 设置Payloads的插入点以及攻击类型（攻击模式）。
3：Payloads 设置payload，配置字典
4：Opetions 此选项卡包含了request headers，request engine，attack results ，grep match，grep_extrack，grep payloads和redirections。你可以发动攻击之前，在主要Intruder的UI上编辑这些选项，大部分设置也可以在攻击时对已在运行的窗口进行修改。

Positions Attack type
Sniper：可以指定多个变量同时进行破解，但只设置一个字典文件，将指定的变量挨个用字典内容进行替换
Battering ram：可以指定多个变量，但仍只设置一个字典文件，将所有的变量一起用字典内容进行替换
Ptichfork：可以指定多个变量，但需要为每个变量分别设置一个字典文件，然后用对应的字典内容对变量同时进行替换
Cluster bomb：指定多个变量，并为每个变量分别设置一个字典文件，然后用字典内容组合对变量进行替换

Payload标签主要用于payload配置
Payload Sets Payload数量类型设置
Payload Opetions[Simple list] 该选项会根据选项1中Payload type的设置而改变
Payload Processing 对生成的Payload进行编码、加密、截取等操作
Payload Encoding 你可以配置哪些有效载荷中的字符应该是URL编码的HTTP请求中的安全传输。任何已配置
的URL编码最后应用，任何有效载荷处理规则执行之后

Repeater 模块：改包

模块功能：可以多次手工 sql 注入，在 http 头，cookie,能提交的参数位置都可以注入；也把 post 请求改为 get 请求

Sequencer 模块：

功能模块：Sequencer 是一种用于分析数据项的一个样本中的随机性质量的工具。你可以用它来测试应用程序的 session tokens(会话 tokens)戒其他重要数据项的本意是不可预测的；

Decoder 模块：编码模块

Comparer 模块：比较模块

模块功能：比较抓到的包有什么不同；有时候大家抓的包内容很多，需要对比的时候这个模块的功能就体现出来了

project Options:可以把日志请求保存下来，得到拦截的日志文件，可以去进一步测试利用

User options 模块：主要用来配置一些常用的选项

Alerts 模块：查看日志
