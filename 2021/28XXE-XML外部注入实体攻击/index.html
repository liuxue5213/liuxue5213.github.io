XXE-XML外部注入實體攻擊

XML外部实体注入攻击(XML External Entity)，漏洞是在对不安全的外部实体数据进行处理时引发的安全问题

XML语法

XML被设计用来传输和存储数据。XML文档是一种树结构，它从”根部”开始，然后扩展到”枝叶”。XML 允许创作者定义自己的标签和自己的文档结构。

语法规则

  1. 所有的XML元素都必须有一个关闭标签
  2. XML标签对大小写敏感
  3. XML必须正确嵌套
  4. XML属性值必须加引号
  5. 在XML中，空格会被保留
  6. 实体引用，不可使用編碼、轉譯的字符

XML数据结构

<?xml version="1.0" encoding="UTF-8"?>
聲明XML版本，編碼格式

XML-DTD

XML-DTD(The document type definition)文档类型定义，是指拥有正确语法的XML被称为“形式良好”合法的XML，可以用來快速检查XML格式

形式良好的XML文档

  1. XML文档必须有根元素
  2. XML文档必须有关闭标签
  3. XML标签对大小写敏感
  4. XML元素必须被正确的嵌套
  5. XML属性必须加引号

XML引用DTD检查的方法有两种

  1. 直接在XML文档中声明并引用

PCDATA是被解析的字符数据（parsed character data）
PCDATA是会被解析器解析的文本,这些文本将被解析器进行检查

  1. 在XML中引入一个外部的DTD文件

XXE注入的思路

  1. 读取POST提交的数据
  2. 通过POST提交XML代码，
  3. XML代码中引用外部DTD，读取想要的系统文件內容
  4. 通过simplexml_load_string()函数显示数据
