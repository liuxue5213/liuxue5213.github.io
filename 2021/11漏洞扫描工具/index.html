漏洞扫描工具

一、 Nessus 主机发现、系统漏洞

Nessus是世界上最流行的漏洞扫描程序，全世界有超过 75000 个组织在使用它。
该工具提供完整的电脑漏洞扫描服务，并随时更新其漏洞数据库。Nessus 不同于传统的漏洞扫描软件，
Nessus 可同时在本机和远端上遥控，进行系统的漏洞分析扫描。

完整的电脑漏洞扫描服务，随时更新漏洞数据库，并且同事支持本机或远程遥控

下载地址 https://www.tenable.com/downloads/nessus?loginAttempted=true

免费版注册后，激活码一次性使用

debian
https://www.tenable.com/downloads/api/v1/public/pages/nessus/downloads/13035/download?i_agree_to_tenable_license_agreement=true

dpkg -i Nessus-8.15.1-debian6_amd64.deb

启动nessus

浏览器输入配置页面地址 https://ip:8834

激活

选择 Managed Scanner，点击 Continue

Managed by 选择 Tenable.s 然后 Continue

输入平台的管理账号和密码 然后submit

先查看安装目录 find / -name nessus

生成查询码
sudo /opt/nessus/sbin/nessuscli fetch –challenge

用浏览器打开：https://plugins.nessus.org/v2/offline.php，输入查询码和激活码，点击 Submit

获得 all-2.0.tar.gz 插件包下载地址

记录下载地址，license证书

更新插件
sudo service nessusd stop
sudo /opt/nessus/sbin/nessuscli update ./all-2.0.tar.gz

记录返回结果中的日期序号 201906031532

到Nessus设置页，点击Activation Code旁边的图标

Offline 输入license证书，点击Activate

如果注册失败，根据提示重新进行操作

破解

创建一个文件，PLUGIN_SET按序号来填写

备份插件目录
cp -r /opt/nessus/lib/nessus/plugins /opt/nessus/lib/nessus/plugins.bak

查找存在的 plugin_feed_info 文件
sudo find /opt -name plugin_feed_info.inc

替换相应的文件
sudo cp plugin_feed_info.inc /opt/nessus/lib/nessus/plugins/
sudo cp plugin_feed_info.inc /opt/nessus/var/nessus/

启动nessus 然后重新等待初始化，进入管理页面
sudo service nessusd start

恢复插件
sudo service nessusd stop

sudo cp -r /opt/nessus/lib/nessus/plugins.bak /opt/nessus/lib/nessus/plugins
sudo cp plugin_feed_info.inc /opt/nessus/lib/nessus/plugins/
sudo cp plugin_feed_info.inc /opt/nessus/var/nessus/

重新启动，并再次初始化
sudo service nessusd start

脚本更新

mac 安裝 nessus

參考鏈接 https://blog.csdn.net/u012949658/article/details/103924141

https://www.tenable.com/downloads/nessus 下載mac版本，然後安裝

啟動服務
cd /Library/Nessus/run/sbin
sudo ./nessusd restart

手動更新
sudo ./nessuscli update

挑战码
sudo ./nessuscli fetch –challenge

輸入挑战码和激活碼 獲取插件安裝包地址和lisence
https://plugins.nessus.org/v2/offline.php

使用插件包更新
sudo ./nessuscli update all-2.0.tar.gz

二、 Nikto 扫描目录

Nikto 是一款开源的（GPL）网页服务器扫描器，它可以对网页服务器进行全面的多种扫描，包含超
过 3300 种有潜在危险的文件 CGIs；超过 625 种服务器版本；超过 230 种特定服务器问题

用于网页服务器扫描，可以发现网站危险文件,扫描目录

查看插件
nikto -list-plugins

nikto -host 192.168.1.1 -port 80

扫描多个地址
nikto -host 编辑的地址文件

扫描ssl
nikto -host ip -port 443 -ssl

保存扫描结果
nikto -host 192.168.1.1 -port 80 -output /root/1.txt

结合nmap自动发现扫描
nmap -p 80 192.168.1.0/24 -port 80 -oG - | nikto -host -
-oG - 将扫描结果输出到 -里面

-evasion 使用 libwhisker 对 ids 的躲避技术，可使用一下类型

  1. 随机url编码 非utf-8
  2. 自选择路径 /./
  3. 过早结束的url
  4. 有限考虑长随机字符串
  5. 参数欺骗
  6. 使用tab作为命令的分隔符
  7. 使用变化的url
  8. 使用Windows路径分隔符 \

nikto -evasion 1234 -host 192.168.1.1

ids intrusion detection system 入侵检测系统
可以发现可疑传输发出警报或者主动反应措施的网络安全设备，积极主动的安全防护技术

显示正在扫描的详细信息，重复操作即可关闭显示

  1. 扫描过程中
  2. 回车 v

d 显示debug信息
e 错误报告
p 进度报告
r 重定向显示
c 显示cookie
a 授权显示
N 下一个ip
p 暂停
q 退出

三、 Skipfish 代码层面、漏洞报告

谷歌创建的Web应用程序安全扫描程序,是一种活跃的Web应用程序安全侦察工具，它通过执行递归爬网和基于字典的探针为目标站点准备交互式站点扫描，该工具生成的最终报告旨在作为与业 Web 应用程序安全评估的基础

cpu占用小，优化过的http处理，c底层，速度快
易于使用，具有自动学习功能，动态词表创建、表单自动完成
高质量，低误报，差异安全检查，

skipfish 在命令行下执行，但结果不會输出到终端，而是保存到某个文件中幵以 html 格式存储

skipfish -o test ip

批量扫描 1.txt
skipfish -o test2 @1.txt

使用字典
skipfish -o test3 -S /usr/share/skipfish/dictionaries/minimal.wl ip

同一个ip 多个站点目录扫描
skipfish -o test4 -I ip

-X 不检查包含string的url
-K 不对指定参数进行fuzz测试
–config 指定配置文件

跨站点扫描 A网站链接B网站
skipfish -o test5 -D www.baidu.com -I /dvwa/ -S /usr/share/skipfish/dictionaries/minimal.wl ip

限制請求的頻率
skipfish -o test -l 10 -m 10 ip
-l 每秒最大请求数
-m 每个ip最大并发

使用密码登录后扫描
skipfish -o test6 –auth-form http://192.168.1.53/dvwa/login.php –auth-user admin –auth-pass password –auth-verify=url http://192.168.1.53/dvwa/index.php -I /dvwa/ http://192.168.1.53

–auth-form 表示登陆账户名密码的界面
–auth-user 后面指定用户名
–auth-pass 后面指定密码
–auth-verify-url 后面指定登陆成功后的界面(即判断是否登陆成功)

四、 AWVS 或者 Appscan

AWVS(Web Vulnerability Scanner)是一个自動化的 Web 应用程序安全测试工具，它可以扫描任何可通过 Web 浏览器访问的和遵循 HTTP/HTTPS 规则的 Web 站点和 Web 应用程序。

适用于任何中小型和大型企业的内联网、外延网和面向客户、雇员、厂商和其它人员的 Web 网站。AWVS 可以通过检查 SQL 注入攻击漏洞、跨站脚本攻击漏洞等来审核 Web 应用程序的安全性。

官网 https://www.acunetix.com

docker安装
pull 拉取下载镜像
docker pull secfa/docker-awvs

将Docker的3443端口映射到物理机的 13443端口
docker run -it -d -p 13443:3443 secfa/docker-awvs

停止启动

容器的相关信息
awvs13 username: admin@admin.com
awvs13 password: Admin123
AWVS版本：13.0.200217097

浏览器访问：https://127.0.0.1:13443/ 即可

修改awvs默认管理员信息
docker ps -a //查看容器，找到相应id
docker exec -it id /bin/bash //进入容器
cd /home/acunetix/.acunetix
./change_credentials.sh

windows 下破解

先安裝awvs

在程序解压目錄下，把 ”license_info.json” 文件放在 ”C:\ProgramData\Acunetix\shared\license” 目彔下，替換原文件；

然後把 license_info.json 文件设置成只读, 右键属性中設置；

把 ”wvsc.exe” 文件，复制到 ” C:\Program Files (x86)\Acunetix\13.0.200205121″ 目錄下，替换原文件；

瀏覽器退出，重新访问：https://127.0.0.1:13443/ 即可；

五、 owasp-zap 掃描

OWASP 是一个开源的、非盈利的全球性安全组织，致力于应用软件的安全研究。其使命是使应用软件更加安全，使企业和组织能够对应用安全风险作出更清晰的决策。目前 OWASP 全球拥有 220 个分部近六万名会员，共同推動安全标准、安全测试工具、安全指导手册等应用安全技术的发展

OWASP ZAP：OWASP Zed Attack Proxy 攻击代理服务器是世界上最受欢迎的免费安全工具之一。ZAP 可以幫助我们在开发和测试应用程序过程中，自動发现 Web 应用程序中的安全漏洞。另外，它也是一款提供给具备丰富经验的渗透测试人员进行人工安全测试的优秀工具

漏洞分析 对系统进行扫描来发现其安全性隐患
渗透测试 对系统进行模拟攻击和分析来确定其安全性漏洞
运行時测试 终端用户对系统进行分析和安全性测试（手工安全性测试分析）
代码审计 通过代码审计分析评估安全性风险（静态测试，评审）

kali上安装 apt-get install zaproxy

然後從 “應用程序中” 搜索 “owasp-zap”
