HTTP简介

  1. HTTP (超文本传输协议)
  2. HTTP设计用来将超文本标记语言(HTML)文档从Web服务器传送到Web浏览器
  3. HTTP是一个请求和回应协议:客户机发送请求，服务器对请求给出回应
  4. HTTP使用可靠的TCP连接,默认TCP端口是80。HTTPS承载于TLS或SSL协议层之上，默认端口为443

HTTP非持久连接

  1. 浏览器首先初始化与运行http服务器主机的TCP连接，使用URL指定端口
  2. 浏览器通过与TCP连接相关联的本地套接字发出一个HTTP请求消息
  3. 服务器接收到请求之后，解析请求并处理请求，然后通过- -个套接字发出相应消息
  4. 服务器通知TCP需要关闭连接(在浏览器收到相应消息之后才会真正终止连接)
  5. 浏览器收到响应消息，TCP连接终止

HTTP持久连接
持久连接没有关闭连接这一步，服务器响应后会继续让TCP连接保持打开状态

HTTP特点

  1. 支持客户/服务器模式
  2. 简单快速，客户向服务器请求服务时，只需传送请求方法和路径
  3. 灵活，HTTP允许传输任意类型的数据对象。正在传输的类型由Content- Type加以标记
  4. 无连接，每次连接只处理一个请求。服务器处理完客户的请求，并收到客户的应答后，即断并连接。
  5. HTTP协议是无状态协议，无状态是指协议对于事务处理没有记忆能力

HTTP报文

  1. HTTP的一个会话，由Request和response组成
  2. HTTP请求(Requests) 由请求行、消息报头、请求正文三个部分组成
    请求行:方法，URL,协议/版本(Method-URl-Protocol/Version)
    消息报头/请求头(Request headers)
    请求正文(Entity body)
  3. HTTP响应(Responses)也包含三个部分:状态行、消息报头、响应正文
    状态行:协议状态代码描叙(Protocol Status code-Description)
    消息报头(Response headers)
    响应正文(Entity body)

设置代理

  1. 根据协议的学习我们了解HTTP协议分为两个部分:请求和回应。
  2. 平时浏览网页，请求报文是经过浏览器包装好的，不通过工具直接访问只能够控制很少的部分。
  3. 要想真正的人为控制与服务器交互，只能通过代理抓包的方法。
  4. 代理:作为客户端和服务器的中间者，在利用http协议交互时，所有请求和回应都不会直接发送给目标而是由代理接受和转发

HTTP请求首部

  1. Host:主要用于指定被请求支援的Internet主机和端口号
  2. User-Agent:向服务端传递客户端操作系统、 浏览器、和其他属性
  3. Referer;包含一个URL，代表当前URL的上一个URL
  4. Cookie:是一段文本，通常来表示请求者的身份
  5. Range:Range可以请求实体部分内容，多线程下载一定会用到此请求头
  6. x-forward-for:即XFF头, 代表请求端的IP，也可以是多个，中间用逗号隔开
  7. Accept:用于指定客户端接收哪些MIME类型的信息
  8. Accept-Charset:用于指定客户端接收的字符集
  9. Content-Type:用于向接收方指示实体的介质类型
  10. Content-Encbding:被用作媒体类型的修饰符,表示了已经被用到实体正文的附加内容编码,想要获得content-type报头域中所弓|用的媒体类型，必须采用相应的解码机制
  11. Content-Length:用于指明实体正文的长度，以字节方式存储的十进制数字来表示
  12. Last-Modified:用于指示资源的最好修改时间和日期

HTTP请求方式

HTTP响应状态码
200 请求成功
302 重定向
400 客户端有语法错误，不能被服务器解析
401 请求未授权
403 服务器收到请求，但是拒绝提供服务
404 请求的资源不存在
500 服务器内部错误
503 服务器当前不能处理客户端请求，过段时间恢复正常

什么是URL

URL是统一资源定位符，对可以从互联网上得到的资源的位置和访问方法的一种简洁的表示，是互联网上标准资源的地址。

互联网上的每个文件都有一个唯一的URL，它包含的信息指出文件的位置以及浏览器应该怎么处理它。

基本URL包含模式（或称协议）、服务器名称（或IP地址）、路径和文件名，
如协议：//授权/路径?查询

完整的、带有授权部分的普通统一资源标志符语法看上去如下：
协议：//用户名:密码@子域名.域名.顶级域名:端口号/目录/文件名.文件后缀?参数=值#标志
