CobaltStrike

CobaltStrike

CobaltStrike是一款渗透测试神器，被业界人称为CS神器。

CobaltStrike分为客户端与服务端，服务端是一个，客户端可以有多个，可被团队进行分布式协团操作。

CobaltStrike 集成了端口转发、服务扫描，自动化溢出，多模式端口监听，windows exe木马生成，windows dll木马生成，java木马生成，office宏病毒生成，木马捆绑。

钓鱼攻击包括：站点克隆，目标信息获取，java执行，浏览器自动攻击等等强大的功能

安装

需要java环境，解压后直接运行加本机ip和客户端连接密码

apt install default-jre

sudo update-alternatives –config java
显示Java的每个安装及其安装路径
vim /etc/environment
末尾追加 JAVA_HOME=”/usr/lib/jvm/java-11-openjdk-amd64/bin/java”
source /etc/environment
echo $JAVA_HOME

开启团队服务

团队服务只可以运行在linux环境下

./teamserver {ip} {password}

scaling red operations

跨越长期时间内的多个目标网络/跨越长期时间内的一个目标网络下的多个行动

目标单元
负责具体目标或具体行动的对象
工作是获得通道、后渗透、内网横向延展
全通道管理单元
给所有网络提供技术通道

最好的通道是没有参与过行动的

payload

Operating System操作系统/Payload/Stager传输器

传输器 stager，一段很小的代码，连接并下载真正的payload到机器，并且加入到内存中

Create a listener

监听类型

  1. beacon
    低且慢、隐蔽的后攻击渗透代理。有两种通信模式，一种是异步通信模式，这种模式通信效率缓慢，Beacon回连团队服务器、下载任务、然后休眠；另一种是交互式通信模式，这种模式的通信是实时发生的。

    http/https 默认设置情况下，HTTP和HTTPS Beacon通过GET请求来下载任务。通过POST请求传回数据
    dns 使用DNS请求将Beacon返回，隐蔽性好，不开放任何端口，规避防火墙，用53端口响应，缺点速度慢

     浏览器的dns缓存->本地host文件->DNS服务器->根DNS服务器

    smb协议是点对点通信

  2. foreign 外部监听器，目的是给其他的payload提供监听，比如msf

监听器管理功能支持重定向。当设置HTTP/HTTPS Beacon监听器时，重定向跳板机IP。
如果提供的Host不属于当前主机（不是团队服务器的IP），CS就假设它是重定向器。

可以使用socat工具将服务器转变为重定向器，将跳板机80端口上的所有连接转发到的团队服务器IP的80端口
跳板机 socat TCP4-LISTEN:80,fork TCP4:[团队服务器]:80

dns beacon

红蓝对抗中，可以通过DNS的方式通信，流量更加隐秘，躲避agent/DLP的检测，实现相对隐秘的渗透方式，反溯源

申请免费域名 http://www.dot.tk/en/index.html?lang=en

  1. 团队服务端，配置域名，开放50050端口和53端口的TCP和UDP 必须
  2. 在域名解析，添加NS记录，主机记录例如dns、dns1、dns2，记录值是当前域名，NS记录就是将解析任务交由一个指定的权威dns服务器(在这里teamserver就是我们的权威dns)
  3. add listener
    => name
    => payload: beacon dns
    => dns hosts: dns.域名、dns1.域名、dns2.域名
    => dns hosts(stager): dns.域名
    => host rotation strategy: round-robin

重定器

创建监听器
CobaltStrike 的内置监听器为Beacon，外置监听器为Foreign
CobaltStrike 的Beacon支持异步通信 和 交互式通信

点击左上方 CobaltStrike 选项——>在下拉框中选择 Listeners ——>在下方弹出区域中单机 add CobaltStrike4.0

目前有以下8种Payload选项：
name：为监听器名字，可任意
payload：payload 类型
HTTP Hosts: shell 反弹的主机，也就是我们 kali 的 ip
HTTP Hosts(Stager): Stager 的马请求下载 payload 的地址
HTTP Port(C2): C2 监听的端口

Malleable C2 是一种特定领域的语言，主要用来控制“Cobalt Strike Beacon”攻击载荷中的网
络指针。

Beacon与teamserver端c2的通信逻辑

  1. stager的beacon会先下载完整的payload执行
  2. beacon进入睡眠状态，结束睡眠状态后用 http-get方式 发送一个metadata(具体发送细节可以
    在malleable_profie文件里的http-get模块进行自定义),metadata内容大概是目标系统的版本，当
    前用户等信息给teamserver端 。如图的 1）
  3. 如果存在待执行的任务，则teamserver上的c2会响应这个metadata发布命令。beacon将会收
    到具体会话内容与一个任务id。
  4. 执行完毕后beacon将回显数据与任务id用post方式发送回team server端的C2(细节可以在
    malleable_profile文件中的http-post部分进行自定义)，然后又会回到睡眠状态。

许多 Beacon 指标由一个 C2 拓展文件控制。一个 C2 拓展文件由设置和数据转换组成。数据转换
是一 个简单的程序，它指定如何转换数据并将其存储在事务中。转换和存储数据的同一程序，向
后解释，还 从事务中提取和恢复数据。

StayKit插件

StayKit 是 Cobalt Strike 持久性的扩展，它利用 SharpStay .NET 程序集的 execute_assembly 函数。攻击者脚本通过读取特定执行类型的模板文件来处理有效负载的创建。

重要提示：要使用脚本，用户只需要加载StayKit.cna攻击者脚本。此外，需要编译SharpStayStayKit.cna程序集并将其放入所在的目录中。最后，如果为有效负载选择模板，有些可能需要使用 Mono 的动态编译。

持久性菜单将添加到信标。由于每种技术的性质不同，因此只有一个 GUI 菜单，没有信标命令。
