Swagger使用的注解及其说明

发布 : 2018-02-15 分类 : PHP 浏览 :

定义接口信息

参数名称 说明
@SWG* @SWG\Post GET/POST/PUT/DELETE 等HTTP 标准请求方法
summary 接口主要摘要
tags 用来给文档分类,name字段必须唯一.某个接口可以指定多个tag,那它就会出现在多组分类中
description 接口描述，支持 Markdown 语法
operationId 操作的 ID，需要唯一
schemes 使用协议 (可以填多种协议) 接口所支持的协议,取值仅限: “http”, “https”, “ws”, “wss”
version 接口版本
host 项目地址, 这个地址会作为每个接口的 url base ,拼接起来一期作为访问地址
consumes 接口默认接收的MIME类型, 有2种方式，x-www-form-urlencoded 和 multipart/form-data 用于接收文件
produces 接口默认的回复MIME类型. api接口用的比较多的就是 application/json 和 application/xml
info 这个里面填写的东西,会放在文档的最开头,用作文档说明
externalDocs 额外的外部文档。
parameters 一个对象来保存参数,可以使用在操作。 这个属性不为所有操作定义全局参数。
responses 返回的对象
deprecated 声明该操作被弃用。 使用声明的操作应该没有。 默认值是false。
security 安全信息的定义

定义请求参数

参数名称 说明
@SWG\Parameter 定义请求参数
name 参数名
format 参数格式 “int32”, “int64”, “float”, “double”, “byte”, “date”, “date-time”
description 参数描述
in 参数来源 “query”, “header”, “path”, “formData”, “body”
body http头承载参数(body只能有一个,有body不能在有其他的)
formData 表单传输
path /users/{id} 作为url的一部分
query /users?role=admin url拼接传输
header X-MyHeader: Value
cookie Cookie: debug=0; csrftoken=BUSe35dohU3O1MZvDCU
type 参数类型. 取值仅限: “string”, “number”, “integer”, “boolean”, “array”, “file”
required 参数是否必须传true
default 参数的默认值
