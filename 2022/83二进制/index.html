二进制

二进制安全

二进制安全是一种主要用于字符串操作函数相关的计算机编程术语，一个二进制安全功能（函数），其本质上将操作输入作为原始的、无任何特殊格式意义的数据流

二进制分析
分析计算机二进制程序（称为二进制文件），及其包含的机器代码和数据属性的科学和艺术。

二进制分析的目标是确定二进制程序的真正属性，以理解它们真正的功能

逆向工程（Reverse Engineering）
即反向工程，是一种技术过程，根据已知的结果，通过逆向分析来推导出程序或软件的具体实现方法，也可以是对一项目标产品进行逆向分析研究，得到产品的设计要素，从而制造出相近但不完全一样的产品。

软件安全

软件逆向工程是通过反汇编和以及调试等手段，分析程序的二进制可执行代码，从而获得程序的算法细节和实现原理。

软件逆向工程的研究对象通常是没有公开源代码的计算机程序，已经经过编译的二进制可执行代码（如 Windows 平台上的 PE 文件，包括 EXE、动态链接库 DLL等文件格式）（Linux 系统的 ELF 文件 以及静态链接库 .so 文件）

白盒测试：已知源代码，代码审计
黑盒测试：逆向、渗透测试

逆向工程类似于软件工程中的“黑盒”测试，需要借助跟踪调试手段，从二进制机器代码开始分析程序。在信息安全领域，逆向工程技术的主要应用包括：
（1）软件破解，破解软件的版权保护措施，让用户不支付授权费用而无限制使
用软件的全部功能；
（2）病毒和恶意程序分析，分析病毒、恶意程序的传播机制和危害并设计出解
决方法；
（3）系统漏洞分析，分析漏洞原理，设计补丁程序或者编写利用程序。
此外，逆向工程技术还用来分析不公开的文件格式、协议等；跨平台硬件驱动程
序的分析和编写；挖掘操作系统未文档化的 API；计算机犯罪取证等

二进制漏洞

二进制漏洞是指程序存在安全缺陷，导致攻击者恶意构造的数据（如 Shellcode）进入程序相关处理代码时，改变程序原定的执行流程，从而实现破坏或获取超出原有的权限

常见的二进制漏洞 主要分为：
溢出攻击：栈溢出、堆溢出、整数溢出等
格式化字符串、双重释放、释放重引用、数组访问越界、内核级、类型混淆、沙盒逃逸以及 PRC 等

原码、反码、补码

加壳

UPX vmp ，就是利用特殊的算法，将 EXE 可执行程序或者 DLL 动态连接库文件的编码进行 改变(比 如实现压缩、加密)，以达到缩小文件体积或者加密程序编码，甚至是躲过杀毒软件查 杀的目的。
目前较常用的壳有 UPX，ASPack、PePack、PECompact、UPack、免疫 007、木⻢彩衣等 等

软件脱壳

顾名思义，就是利用相应的工具，把在软件“外面”起保护作用的“壳”程序去除，还文件本来面 目，这样再修改文件内容或进行分析检测就容易多了

免杀

通过加壳、加密、修改特征码、加花指令等等技术来修改程序，使其逃过杀毒软件的查杀

花指令

通过加入不影响程序功能的多余汇编指令，使得杀毒软件不能正常的判断病毒文件的构造。说 通俗点就是“杀毒软件是从头到脚按顺序来识别病毒。如果我们把病毒的头和脚颠倒位置，杀 毒软件就找不到病毒了

Rootkit

Rootkit 是攻击者用来隐藏自己的行踪和保留 root(根权限，可以理解成 WINDOWS 下的 system 或者管理员权限)访问权限的工具。通常，攻击者通过远程攻击的方式获得 root 访问 权限，或者是先使用密码猜解(破解) 的方式获得对系统的普通访问权限，进入系统后，再通过 对方系统存在的安全漏洞获得系统 的 root 或 system 权限。然后，攻击者就会在对方的系统 中安装 Rootkit，以达到自己⻓久控制对方的目的，Rootkit 功能上与木⻢和后⻔很类似，但远 比它们要隐蔽

模糊测试 (fuzz testing, fuzzing) 是一种软件测试技术。其核心思想是将自动或半自动生成的随机数据输入到一个程序中，并监

视程序异常，如崩溃，断言(assertion)失败，以发现可能的程序错误，比如内存泄漏。

汇编语言

汇编语言(Assembly Language)是任何一种用于电子计算机、微处理器、微控制器或其他可 编程器件的低级语言，亦称为符号语言。 Arm，x86,mips
