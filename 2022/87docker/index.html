docker

简介

Docker是一个开源的应用容器，让开发者可以打包应用以及依赖包到一个可移植的容器中,然后发布到任何流行的Linux机器或Windows机器上,可以实现虚拟化,容器是完全使用沙箱机制, 相互之间不会有任何接口

传统虚拟机内部不但包括完整的应用和其依赖，还包括完整的操作系统，但是容器只包括应用和其依赖，资源占用大大减少，虚拟机对内存和CPU的消耗很大，所以同一配置的服务器，使用容器可以提供更多的服务能力

docker：将应用程序自动部署到容器，使用Go语言编写，在容器执行环境中增加了一个应用程序部署引擎，该引擎的目标是提供轻量快速的环境，可以运行开发者的环境，方便高效的从开发者环境部署到测试环境，然后到生产环境

特点

简单轻量的建模方式，启动非常快，去除了管理程序的开销，容器拥有非常高的性能，同一台宿主机
器中运行更多的容器，用户尽可能的使用系统资源

职责分离，使用docker只需要关心容器中运行的应用程序，而运维人员只需要关系如何管理容器，加强
开发人员写代码的环境和部署的生产环境一致性

缩短程序从开发、测试、上线的周期，让应用程序可以执行，在容器中开发，并在容器中分发，开发、测
试、上线都使用相同的环境，避免额外的调试和部署的开销，有效缩短产品的上线周期

鼓励面向服务的架构，docker推荐单个容器只允许一个程序或者进程，这样就形成了一个分布式的应用程序模型，应用程序或者服务，都变为一系列内部互联的容器，分布式应用程序的调试变得简单，高内聚，低耦合，单一任务，避免在同一服务器上部署不同服务可能带来的相互影响，出现问题也更容易定位问题出现的位置

使用场景

开发、测试、部署服务
创建隔离的运行环境
搭建测试环境
构建多用户的平台及服务（PaaS）基础设施
构建多用户的软件及服务（SaaS）基础设施
高性能、超大规模的宿主机部署

基本组成

Docker Client客户端
客户端会向服务器端也就是守护进程发送请求，守护进程处理完所有的工作并返回结果，客户端向服务器
的访问既可以是本地，也可以是远程

Docker Daemin守护进程

Docker Image镜像 构建打包阶段
容器基于镜像启动和运行，保存了用于启动容器的各种条件，镜像是一个层叠的只读文件系统，最低端是引导文件系统(bootfs)，用户几乎永远不会和引导文件系统发生交互，当一个容器启动后会被移到内存中，而引导文件系统会被卸载，第二层是 rootfs，可以使一种或者多种操作系统，rootfs 永远是只读状态，之后联合加载技术(union mount)，加载若干个文件系统，联合加载是指一次加载多个，在外面看来值加载了一个，最终的文件系统会包括所有的底层文件和目录，一个镜像可以放到另外一个镜像的底部底部称为基础镜像

Docker Container容器 启动和执行阶段
启动的时候docker会在镜像文件系统最顶层加载一个读写文件系统，执行的程序就是在这一层执行，第一次启动容器时，初始的读写层是空的，当文件系统发生变换的时候都会用到这一层，当修改一个文件的时候，首先将这个只读文件复制到这个可读写层，该文件的只读版本依然存在，但是已经被读写层的父版本隐藏，这就是 docker 中写时复制(copy on write)技术，每个只读层都是只读的，并且以后都不会变化，创建一个新容器的时候，docker 会创建一个镜像栈

Docker Registry仓库

安装

curl -fsSL http://mirrors.zju.edu.cn/docker-ce/linux/debian/gpg | sudo apt-key add -
echo ‘deb http://mirrors.zju.edu.cn/docker-ce/linux/debian/ buster stable’ | sudo tee /etc/apt/sources.list.d/docker.list

curl -fsSL https://mirrors.tuna.tsinghua.edu.cn/docker-ce/linux/debian/gpg | sudo apt-key add -
配置 Docker APT repository
echo ‘deb https://mirrors.tuna.tsinghua.edu.cn/docker-ce/linux/debian/ buster stable’ | sudo tee /etc/apt/sources.list.d/docker.list

更新软件、卸载旧版本
sudo apt-get update
sudo apt-get remove docker docker-engine docker.io

安装
sudo apt-get install docker-ce

查看状态
systemctl status docker

启动
systemctl start docker

命令

版本
docker version

详细信息
docker info

搜索镜像
docker search 关键词

下载镜像
docker pull 镜像名

查看容器日志
docker logs -f 容器id或name

查看已下载的镜像列表
docker image

本地所有镜像
docker image -a

显示镜像的摘要信息
docker images –digests

显示完整的镜像信息
docker images –no-trunc

创建并使用容器 以别名运行容器
docker run -i -t –name 容器名 镜像名或镜像ID /bin/bash

退出容器
先按住Ctrl+P、然后按住Ctrl+Q

查看当前运行的容器
docker ps

查看当前所有的容器
docker ps -a

连接容器
docker exec -it 容器名或容器ID /bin/bash

docker start 容器名或容器ID

docker restart 容器名或容器ID

docker stop 容器名或容器ID

停止所有容器
docker stop $(docker ps -aq)

删除容器
docker rm 容器名或容器ID

删除镜像
docker rmi 镜像名或容器ID

端口映射，部署一个容器，并将 80 端口映射到宿主机的 8000 端口上

可以使用–name自定义部署的容器名
docker run -d -p 8000:80 –name 容器名 镜像名

查看容器日志
docker logs [参数] 容器名
-f 追踪日志变化,默认fasle
-t 返回结果加上时间戳
–tail 返回日志结尾,默认false

也可以直接通过镜像部署
docker run -d -p 8000:80 镜像名
-i 全新的
-t 打开登陆终端
-d 后台启动
-p 端口：容器端口
-name 指定名称
–privileged 超级特权模式
