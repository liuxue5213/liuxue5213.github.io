88搭建rmtp服务端

发布 : 2022-09-15 分类 : linux 浏览 :

漏洞扫描工具

docker

CentOS
sudo yum install docker

Ubuntu
sudo apt-get install docker.io

sudo systemctl start docker
systemctl status docker
systemctl enable docker

sudo docker pull tiangolo/nginx-rtmp
sudo docker run -d -p 1935:1935 –name nginx-rtmp tiangolo/nginx-rtmp

防火墙打开1935端口
推流的地址 rtmp://电脑ip地址/live 密钥随便填写

服务器安装

nginx-rtmp-module

wget http://nginx.org/download/nginx-1.8.1.tar.gz
tar -zxf nginx-1.8.1.tar.gz

sudo apt-get update
sudo apt-get install libpcre3 libpcre3-dev
sudo apt-get install openssl libssl-dev

git clone https://github.com/arut/nginx-rtmp-module.git
cd nginx-1.8.1
./configure –add-module=../nginx-rtmp-module
make
sudo make install

cd /usr/local/nginx
sudo ./sbin/nginx

点播视频服务器的配置

修改 nginx.conf 文件

发送测试用的视频文件到服务器

重启服务 sudo ./sbin/nginx -s reload

scp test.mp4 ubuntu@ip:/home/ubuntu/Videos

打开视频播放软件选用的是VLC media-> open network stream….
填写节目地址 rtmp://ip/vod/test.mp4

点击play就可以播放了，不使用RTMP插件nginx自身也是可以实现点播服务的(配置文件的location部分)

直播视频服务器的配置

一共2个位置

  1. 就是给RTMP服务添加一个application这个名字可以任意起;如果打算弄多个频道的直播就可以 live_cctv1、live_cctv2名字任意。

  2. 第二处就是添加两个location字段。

重启服务 sudo ./sbin/nginx -s reload

rtmp://ip/live/test

播放地址 rtmp://ip/live/test

推流ffmpeg

ffmpeg -i wz.mp4 -f flv rtmp://192.168.1.1/live/test
