网络安全(一)渗透测试行业术语

渗透测试行业术语

1. 肉鸡

比喻可以随意被我们控制的电脑

2. 木马

伪装成正常程序的恶意程序，一旦运行可以控制电脑

3. 远控

远距离控制另外一台电脑

4.网页木马

伪装成正常页面的恶意页面，一旦访问便会下载木马自动执行

5. 黑页

攻击者在目标网站放的嘲讽页面

6. 挂马

将木马写进目标网页文件中，让访问者中马

7. 大马

功能强大的网页后门，能执行命令，操作文件，连接数据库

8. 小马

功能单一，一般用来传输大马

9. 一句话后门

对网站进行控制的一段代码

10. 后门

相当于控制目标主机的备用钥匙

11. 拖库

从数据库导出数据

12. 社工库

包含很多隐私信息的库

13. 撞库

用已有数据库进行攻击

14. 提权

提升在服务器中的权限

15. 网络钓鱼

用欺骗性的电子邮件或者伪造的站点来进行诈骗活动。

16. 社会工程学攻击

通过对被攻击者的心理弱点、本能反应、好奇心、信任、贪婪等心理陷阱所采取的欺骗、伤害等手段

17. rookit

攻击者用来隐藏自己的行踪和保留root访问权限的工具

18. IPC

是共享“命名管道”的资源，可通过验证用户名和密码获得相应的权限，在远程管理计算机和查看计算机的共享资源时使用。

19. 弱口令

容易被猜解的密码

20. 默认共享

WINDOWS2000/XP/2003 系统开启共享服务时自动开启所有硬盘的共享

21. shell

命令执行环境

22. 交互式shell

shell等待输入，并且执行输入的命令

23. webshell

以asp、php、jsp或者cgi等网页文件形式存在的一种命令执行环境。

24. 溢出

程序对接受的输入数据没有执行有效的检测而导致错误，造成程序崩溃或者执行攻击者的命令。有堆溢出栈溢出。

25. 注入

用户提交的数据被当做指令进行处理

26. 注入点

类似于回显点

27. 旁站入侵

同一服务器上有多个站点，通过入侵其中一个站点，通过提权夸目录访问其他站点

28. C段扫描

拿下同一网段的服务器，再以该服务器为跳板，去拿下其他同网段内的服务器

29. 内网

局域网。如果IP地址是10.0.0.0—10.255.255.255，172.16.0.0—172.31.255.255，192.168.0.0—192.168.255.255

30. 外网

直接连入Internet，可以与互联网上任意一台电脑互相访问。

31. 中间人攻击

MITM攻击，以中间人劫持的方式进行攻击

32. 端口

相当于数据的传输通道，用于接受数据，然后传输给相应的服务。一般一个端口对应一个服务。

33. 免杀

逃过杀毒软件的查杀

34. 加壳

利用特殊算法，将EXE可执行程序或者DDL动态连接库文件的编码进行改变（比如实现压缩、加密），以达到缩小文件体积或者加密程序编码，甚至是躲过杀毒软件查杀的目的。还可以保护软件不被非法修改或反编译等。

35. 花指令

几句汇编指令，让汇编语句进行一些跳转，使得杀毒软件不能正常的判断病毒文件的构造

36. TCP/IP

网络通信协议，规范了网络上所有的通信设备之间的通信数据格式。

37. 路由器

将IP数据包正确的送到目的地的网络设备，因此也叫IP路由器

38. 蜜罐

好比是情报收集系统，故意让人来攻击，因此来收集黑客的技术手段、所用工具、以及掌握他们的社交

39. 拒绝服务攻击

DDOS！

40. CC攻击

攻击者借助代理服务器生成指向受害主机的合法请求，实现DDOS的伪装

41. 脚本注入攻击

脚本就是漏洞类型，比如SQL注入攻击

42. 加密技术

保密手段，利用技术手段将重要数据转换成乱码

43. 局域网内部的ARP攻击

ARP（地址解析协议），作用是通过目标设备的IP地址，来查询目标设备的MAC地址，以保证通信的进行。攻击者向对方计算机不断发送有欺诈性质的ARP数据包，数据包内包含有与当前设备重复的MAC地址，使对方在回应报文时，由于简单的地址重复错误而导致不能进行正常的网络通
信。容易造成目标无法正常上网，以及弹框“本机的XX段硬件地址与网络中的XX段地址冲突”

44. 欺骗攻击，有哪些攻击方式

网络欺骗技术主要有：HONEYPOT和分布式HONEYPOT 、欺骗空间技术等。

主要方式有：IP欺骗、ARP欺骗、DNS欺骗、Web欺骗、电子邮件欺骗、源路由欺骗（通过指定路由，以假冒身份与其他主机进行合法通信或发送假报文，使受攻击主机出现错误动作）、地址欺骗（包括伪造源地址和伪造中间站点）等。

45. 嗅探

捕获在网络中传输的数据信息

46. 跳板

一个具有辅助作用的机器，利用该机器控制其他主机，一般和肉鸡连用

47. 权限

控制用户可以访问的内容

48. IP地址

相当于电脑的身份证，长度32位，由4个数值部分组成，每个数值部分可取值0-255，各部分之间用“-”分开。

49. RARP反向地址解析协议

将硬件地址映射到网络地址

50. UDP 用户数据报协议

无连接的传输层协议，提供面向事务的简单不可靠信息传送服务

51. TCP协议

面向连接的、可靠的、基于字节流的传输层通信协议。SYN->SYN-ACK->ACK

52. FTP文件传输协议

允许用户以文件操作的方式（文件的增、删、改、查、传送等）与另一主机相互通信。

53. SMTP简单邮件传送协议

系统之间传送电子邮件

54. TELNET终端协议

允许用户以虚拟终端的方式访问远程主机

55. HTTP协议

简单的请求—响应协议，运行在TCP之上，指定了客户端可能发送给服务器什么样的消息以及得到什么样的响应。

56. HTTPS安全超文本传输协议

加密的HTTP协议

57. TFTP

简单文件传输协议

58. ICMP协议

Inter控制消息协议。ping的时候用此协议。操作系统规定ICMP数据包最大不超过64KB，而如果发送的ICMP包超过64KB，就会导致主机出现内存分配错误，进而导致耗费大量的资源处理，疲于奔命，最终瘫痪，死机。

59. DNS协议

将域名解析成IP，也可以将IP转换成域名

60. root

最高权限的用户，也就是超级管理员

61. EXP/Exploit

漏洞利用代码，运行之后对目标进行攻击

62. POC/Proof of Concept

漏洞验证代码，检测目标是否存在对应漏洞的一个程序而已。

63. Payload

指“成功”exploit之后，真正在目标系统执行的代码或指令。

64. ShellCode

shell代码，是Payload的一种，由于其正向/反向shell而得名。

65. 脱壳

将保护软件的壳脱掉，方便修改

66. 蠕虫病毒

通常是一个脚本程序能调用功能更大的组件来完成自己的功能，比如XSS蠕虫。

67. LAN

局域网，LAN上所有的网络属于同一个内网

68. Proxy

代理

69. HTML

编写网页的计算机语言

70. CSS

美化网页内容的作用

71. Javascript

增加网页功能

72. CMS

网页模板

73. 独立服务器

独立服务器整体硬件都是独立的，可以部署任何想要的东西

74. VPS

操作系统虚拟化技术，多用于虚拟机。

75. 域名

IP地址的别称

76. CTF（夺旗赛）

网络安全技术人员之间技术竞技的一种比赛形式

77. awd攻防对抗赛

分为攻击者和防守者，攻者得分，失者扣分。也就是攻击者攻击别人的靶机获得Flag可以得分，被攻击的要扣分，在攻击的同时也要保护好自己的主机不被别人得分，以防扣分。

78. CVE

漏洞披露

79. CNVD

国家信息安全漏洞共享平台

80. 0day

还未被公布的或者还没有相应补丁的漏洞

81. 1day

刚发布的漏洞，虽然官网有补丁，但是依旧有很多主机没安装补丁的漏洞

82. Nday

已经公布出来的0day

83. APT攻击

高级可持续性攻击，指组织或者小团体利用先进的技术手段对特定目标进行长期可持续经网络攻击的攻击形式。

84. 渗透测试

黑盒测试、白盒测试、灰盒测试

85. 暗网

隐藏的网络，需要使用特定的软件、配置或者授权才能登录。一般用Tor洋葱浏览器。暗网是利用加密传输、P2P对等网络、多点中继混淆等，为用户提供匿名的互联网信息访问的一类技术手段。非常的匿名。

86. 恶意软件

可以达到非授权控制计算机或窃取计算机数据等多种恶意行为的程序。

87. 间谍软件

在用户不知情的情况下，在其电脑、手机上安装后门，具备手机用户信息、监听、偷拍的软件

88. 洪水攻击

攻击者对网络资源发送过量数据时就发生了洪水攻击，只要攻击流量足够大，就可以将防御手段打穿。DDOS也属于一种。

89. SYN攻击

利用操作系统TCP协议设计上的问题执行的DDOS，涉及TCP建立连接时三次握手

90. DoS攻击

拒绝服务攻击。攻击者通过利用漏洞或发送大量的请求导致攻击对象无法访问网站或网络无法被访问。

91. DDoS

分布式DoS攻击，通常是多个肉鸡一起向你发送网络请求，导致网络阻塞而不能正常上网。

92. 抓鸡

设法控制其电脑，将其沦为肉鸡

93. 端口扫描

发送一组端口扫描信息，通过该信息了解到可以从哪里探寻到攻击弱点，并了解其提供的计算机网络服务类型，试图以此入侵某计算机

94. 反弹端口

让被控端主动连接控制端

95. 鱼叉攻击

让欺骗性的电子邮件看起来更加可信的网络钓鱼攻击。

96. 钓鲸攻击

针对高级管理人员和组织内其他高级人员的网络钓鱼攻击

97. 水坑攻击

先分析目标的上网活动规律，寻找攻击目标经常访问网站的弱点，先将此网站植入攻击代码，一旦攻击目标访问该网站就会“中招”

98. C2（Command and Control）

命令行控制常见于APT攻击场景中，攻击者与被攻击目标机器上恶意软件进行交互。

99. 供应链攻击

黑客攻击目标机构的合作伙伴，并以该合作伙伴为跳板，达到渗透目标用户的目的。类似水坑攻击。

100. 渗透

通过扫描检测网络设备及系统有么有安全漏洞，有的话就入侵，就像一滴水透过一块有漏洞的木板，渗透成功就是入侵。

101. 横移

攻击者入侵后，从目标机器向内部网络进行拓展，搜寻控制更多的系统

102. 暗链

看不见的网站链接，类似于友情链接，有效提高网站权重。

103. 暴库

通过恶意代码让网站爆出其他一些敏感数据

104. 薅羊毛

指网赚一族利用各种网络金融产品或红包推广线下抽成赚钱。

105. BEC（商业电子邮件攻击）

针对高管人员的攻击，攻击者通常冒充决策者的邮件，来说服/诱骗高管短时间完成经济交易。

106. 电信诈骗

通过电话、网络和短信方式，编造虚假信息，设置骗局，诱使受害人打款或转账，对受害人实施的犯罪行为。

107. 杀猪盘

属于电信诈骗的一种，是一种网络交友诱导股票投资、赌博等类型的诈骗，指放长线“养猪”诈骗，养的越久，诈骗得越狠。

108. 黑产

以互联网作为媒介，以网络技术为主要手段，来进行的非法交易。比如非法的数据交易产业。

109. 黑帽黑客

以非法目的进行黑客攻击的人，即无授权的渗透。

110. 白帽黑客

有授权的渗透攻击

111. 红帽黑客

正义的白帽黑客 ，主要利用自己掌握的技术去维护国内网络安全的人。

112. 红队

攻防演习的攻击队伍

113. 蓝队

攻防演习的防守队伍

114. 紫队

攻防演习的监理方或裁判

115. 加密机

主机加密的设备，与主机之间使用的是TCP/IP协议通信。

116. CA证书

为实现双方安全通信提供了电子认证

117. SSL证书

数字证书的一种，类似于驾驶证的副本，因为配置在服务器上，也称为SSL服务器证书

118. 防火墙

保护计算机的安全，可以有选择性的接受外部访问。

119. IDS

入侵检测系统，对黑客发起的攻击或者攻击之前就检测，并加以拦截。IDS不同于防火墙，防火墙只能屏蔽入侵，而IDS却能在入侵发生之前就作出反应。

120. NIDS

网络入侵检测系统，主要用于检测Hacker或Cracker

121. IPS

入侵防御系统，一般作为防火墙和防病毒软件的补充来投入使用。

122. 杀毒软件

用于保护电脑，防御病毒的软件

123. 反病毒引擎

一套判断特定程序的行为是否为病毒程序（包括可疑的）的技术机制

124. 防毒墙

类似杀毒软件，主要部署在网络出口，用于对病毒进行扫描和查杀。

125. 告警

指网络安全设备对攻击行为产生的警报

126. 误报

无效告警

127. 漏报

检测出非法行为但是没有告警

128. NAC

网络准入控制，宗旨是防止病毒和蠕虫等新兴黑客技术对企业安全造成危害

129. 漏扫

漏洞扫描，基于漏洞数据库的扫描，从而发现可利用漏洞的一种安全检测行为。

130. UTM

统一威胁管理，即将不同的设备的安全能力集中在同一网关上，实现统一管理和运维。

131. 网闸

连接两个独立主机系统的信息安全设备。其中以数据文件形式进行的无协议摆渡。

132. 堡垒机

运用各种技术手段监控和记录运维人员对网络内的服务器、网络设备、安全设备、数据库等设备的操作行为，以便集中报警、及时处理及审计定责。

133. 数据库审计

实时记录网络上的数据库活动，对数据遭受的攻击进行告警，对攻击行为进行阻断等。

134. DLP

数据防泄露

135. VPN

虚拟专用网，通过对数据包的加密和数据包目标地址的转换实现远程访问

136. SD—WAN

软件定义广域网，用于连接企业网络、数据中心、互联网应用及云服务

137. 网关

通常指路由器、防火墙、IDS、VPN等边界网络设备

138. WAF

Web应用防护墙

139. SOC

安全运行中心，协助管理员进行时间分析、风险分析、预警管理和应急响应处理的安全管理系统。

140. LAS

日志审计系统，主要功能是提供日志收集、检索和分析能力。

141. NOC

网络操作中心或网络运行中心，是远程网络通讯的管理、监视和维护中心。

142. SIEM

安全信息和事件管理

143. 上网行为管理

帮助互联网用户控制和管理对互联网使用的设备。

144. 沙箱

一个用于安全运行程序的机制，常常用来运行不可信的程序。

145. 沙箱逃逸

一种识别沙箱环境、并利用静默、欺骗等技术，绕过沙箱检测的现象。

146. 网络靶场

模拟真实环境的平台

147. 加密技术

算法和秘钥

148. 黑名单

在名单上的软件、地址等，都被认为是违法的、不可访问的。

149. 白名单

在名单上的软件、地址等，都被认为是合法的、可访问的。

150. 边界防御

以网络边界为核心的防御模式，将所有的安全威胁都挡在外网。

151. 南北向流量

指数据中心内外部通信所产生的流量

152. 东西向流量

指数据中心内部不同主机之间通信所产生的流量

153. 规则库

类似于黑白名单，用于存储大量安全规则。也可以比喻为网络空间的法律 。

154. 下一代

用于表示产品或者技术有较大的幅度的创新

155. 大数据安全分析

区别于传统的被动规则匹配的防御模式，以主动收集和分析大数据的方法，找出其可能存在的安全威胁。因此也称为数据驱动安全。

156. EPP

端点保护平台，部署在终端设备上的安全防护解决方案，用户防止针对终端的恶意软件、恶意脚本等安全威胁。

157. EDR

端点检测与响应，对操作系统调用等异常行为分析，检测和防护未知威胁，最终达到杀毒软件无法解决未知威胁的目的。

158. NDR

网络检测与响应，可提高网络安全的可见性和威胁免疫力。

159. 安全可视化

指在网络安全领域中的呈现技术，将网络安全加固、检测、防御、响应等过程中的数据和结果进行可视化

160. NTA

网络流量分析

161. MDR

托管检测与响应，主要用来监控网络

162. 应急响应

指一个组织为了应对各种意外事件发生所做的准备以及在事件发生后所采取的措施

163. XDR

指以检测和响应技术为核心的网络安全策略的统称，包括EDR、NDR、MDR等

164. 安全运营

贯穿产品研发、业务运行、漏洞修复、防护与检测、应急响应等一系列环节的管理方法和流程，保障业务安全。

165. 威胁情报

对目标有产生威胁的情报，主要有人读情报和机读情报

166. TTP

战术Tactics、技术Techniques和过程Procedures，作为威胁情报的一种组成部分，可为安全分析人员提供决策支撑。

167. IOC

失陷标示，用以发现APT团伙，木马后门，僵尸网络控制的标示。

168. 态势感知

基于环境、动态、整体的洞悉安全风险的能力

169. 探针

网络安全探针，部署在网路拓扑的关键节点上，用于收集和分析流量和日志，发现异常行为等

170. 网络空间测绘

用搜素引擎技术来提供交互，让人们可以方便的搜索到网络空间上的设备。

171. 逆向

从产品中提取原理及设计信息并应用于再造及改进的行为都属于逆向工程。在网络安全中，更多的是调查取证，恶意软件分析等。

172. 爬虫

按照一定规则，自动抓取万维网信息的程序或脚本

173. 防爬

防止信息被爬虫获取

174. 区块链

共享数据库，存储于其中的数据或信息，具有“不可伪造”、“全程留痕”、“可以追述”、“公开透明”、“集体维护”等特征

175. 风控

利用大数据分析的方法来判断业务可能存在的安全风险。主要用于金融信贷领域，防止坏账的发生。

176. 渗透测试

为了证明网络防御按照预期计划正常运行而提供的一种机制。通常是邀请攻击团队，按照一定规则攻击既定目标，从而找出其中存在的漏洞或其他安全隐患，并出具测试报告和整改建议

177. 代码审计

根据源代码去寻找漏洞

178. NTLM验证

一种身份验证。

179. 数据脱敏

对某些敏感信息通过脱敏规则进行数据的变形，实现敏感隐私数据的可靠保护。

180. CISO

首席信息安全官，是主要安全的负责人
